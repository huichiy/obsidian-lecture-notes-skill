# Vault context for Claude Code

Read this before running tools in this directory. It primes you with what's here, what conventions are in use, and what *not* to touch.

## What this directory is

This is an **Obsidian vault**, not a software project. It contains lecture notes, atomic concept notes, MOC pages, canvases, and Bases trackers, mostly organized per subject. It is a **git repo** (auto-committed via the `obsidian-git` plugin) — so any changes you make get versioned.

## The lecture-notes skill is installed

The user has the **`lecture-notes`** Claude Code skill at `~/.claude/skills/lecture-notes/`. If they upload a PDF / PPT / slide deck or ask for note generation, lint, canvas, revision pack, tracker, or atomic extraction — that skill auto-triggers. Skill source repo: `~/Documents/obsidian-lecture-notes-skill/` (separate from this vault).

## File-naming conventions

- **Lecture notes**: `{SUBJECT} Lec {NN} — {Topic}.md` (en-dash `—`, not hyphen). Zero-padded lecture number when ≥ 10. Example: `DSF Lec 04 — Exploratory Data Analysis.md`.
- **Atomic concepts**: `{Concept Name}.md` in `{Subject}/Concepts/` (Title Case, single space). Example: `Box Plot.md`.
- **MOC**: `{SUBJECT} - MOC.md` (regular hyphen, single spaces). Example: `DSF - MOC.md`.
- **Tracker**: `{SUBJECT} - Tracker.base`.
- **Canvas**: `{SUBJECT} - Knowledge Map.canvas` or `{SUBJECT} - Exam Map.canvas` in `{Subject}/Canvas/`.
- **Revision pack**: `{SUBJECT} - Revision Pack ({range}).md` in `{Subject}/Revision/`.

## The `(new)` suffix convention

Files like `DSF Lec 04 — Exploratory Data Analysis (new).md` are **upgraded versions** generated by the v1.1+ skill, living in `Test/` while the user decides whether to replace the older same-name file in `DSF Note/`. **Don't delete `(new)` files** or treat them as duplicates — they're intentional. When the user migrates, the `(new)` suffix is dropped and the old file is overwritten.

## `Test/` is a sandbox

`Test/` contains a complete end-to-end demo: 4 new DSF lectures, 28 atomic concepts, 2 canvases, 2 revision packs. Used to validate skill output before migrating to main subject folders. Safe to modify; don't mistake it for production notes.

## Folders to leave alone

- **`Archieve/`** (sic — preserve the spelling, it's been like that): cold storage for old projects / drafts.
- **`Excalidraw/`**: Excalidraw plugin's storage. Don't edit `.excalidraw` files via text tools.
- **`.obsidian/`**: Obsidian config. Modify only via the app's settings UI, **except** for `.obsidian/snippets/` (CSS files are fine to edit).

## Plugins installed (relevant ones)

The vault has these enabled — you can assume their features work:

- **Advanced Canvas** — supports `styleAttributes.path = "dotted"` for weak edges in `.canvas` files.
- **Bases** (built-in, Obsidian 1.9+) — renders `.base` files as database views.
- **Dataview** — used by the MOC template's auto-populating query blocks.
- **Style Settings** — exposes CSS variables in Settings UI; the `concept-levels.css` snippet uses this.
- **Templater** — supports template insertion.
- **obsidian-git** — auto-backups; commit messages look like `vault backup: YYYY-MM-DD HH:MM:SS - N files`.
- **Excalidraw** — for hand-drawn diagrams.

## CSS snippet active

`.obsidian/snippets/concept-levels.css` is installed and toggled ON. Atomic concept notes can use `#level/1` (purple pill), `#level/2` (blue pill), `#level/3` (red pill) tags — render in Reading view.

## YAML schema (lecture-notes)

Lecture notes follow this YAML schema. Don't break it.

```yaml
---
subject: DSF                # must match a code from the table above
course_code: CDS6214
lecture: 4                  # integer
title: "..."                # always quoted
source: Lec04 EDA.pdf
type: lecture-notes
created: YYYY-MM-DD
tags:
  - dsf
  - lecture-04
  - revision
  - exam-prep
aliases:
  - "DSF Lec 4"
  - "CDS6214 Lecture 4"
related:
  - "[[Some Concept]]"
exam_weight: low | medium | high
status: draft | reviewed | complete
---
```

Atomic concepts use `type: concept`, plus `level: 1|2|3` and `parent: "[[exact lecture filename]]"`.

## Don't do these

- ❌ Don't manually edit auto-generated lecture notes for content — re-run the skill instead.
- ❌ Don't rename lectures to drop the en-dash (`—`) — many backlinks point to the exact form.
- ❌ Don't auto-fix lint findings without showing the user a diff preview first (the LINT mode handles this; if you're doing it manually, follow the same pattern).
- ❌ Don't push to remote on the user's behalf unless explicitly asked. `obsidian-git` may auto-commit, but pushing is a deliberate action.
